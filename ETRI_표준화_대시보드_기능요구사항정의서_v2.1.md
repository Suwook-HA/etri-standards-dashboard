## 1. 개요

### 1.1 목적

ETRI 표준연구본부에서 수집한 표준화 현황 데이터(650건)와 표준체계 및 선도전략 보고서의 분석 결과를 하나의 인터랙티브 웹 대시보드로 제공한다. 사용자는 요약 수치를 클릭하면 해당 표준 목록으로 즉시 이동하며, 다양한 분류 기준 간 자유롭게 탐색할 수 있다.

### 1.2 배경

현재 표준화 현황 분석 결과는 Word 보고서와 Excel 원본 데이터로 분리되어 있어, 특정 수치의 근거가 되는 개별 표준을 확인하기 위해 별도 작업이 필요하다. 이를 해결하기 위해 요약 지표 → 세부 목록 → 개별 표준 상세의 3단계 드릴다운 네비게이션을 갖춘 통합 대시보드를 구축한다.

### 1.3 범위

- **데이터 소스**: 표준화 현황 전수조사 엑셀 파일 (650건, 27개 컬럼)
- **참조 문서**: ETRI 표준체계 및 선도전략 2025 보고서
- **데이터 갱신**: 연 1회 전수조사 엑셀 파일 교체 방식 (담당자 직접 업로드)
- **지원 환경**: 웹 브라우저 (Chrome, Edge 최신 버전)
- **사용 언어**: 한국어

### 1.4 데이터 갱신 운영 방식

매년 표준연구본부에서 전수조사를 완료한 뒤, 담당자가 갱신된 엑셀 파일을 대시보드에 직접 업로드한다. 시스템은 이전 버전과 비교하여 변경사항(신규·수정·삭제)을 자동으로 감지하고, 담당자가 내용을 확인한 뒤 최종 적용하는 2단계 승인 방식을 따른다. 적용 전 언제든 이전 버전으로 롤백할 수 있다.

```
[담당자 엑셀 업로드]
        ↓
[자동 유효성 검사 + 변경사항 감지]
        ↓
[변경 미리보기 화면 — 담당자 확인]
        ↓
[적용 확정] → [대시보드 즉시 반영 + 버전 스냅샷 저장]
```

## 3. 시스템 구조 개요

```
[대시보드 홈]
    │
    ├─ [전략기술 뷰]     12대 기술별 현황 비교
    ├─ [표준기구 뷰]     기구별 참여 현황
    ├─ [부서 뷰]         직할부서·본부별 현황
    ├─ [연도별 뷰]       연도 추이 및 파이프라인
    └─ [표준특허 뷰]     특허 확보 현황
    
각 뷰의 수치 클릭 → [표준 목록 패널] → 개별 항목 클릭 → [표준 상세 패널]
```

## 5. 뷰별 기능 요구사항

### 5.1 홈 대시보드 뷰 (기본 화면)

전체 현황을 한눈에 파악하는 오버뷰 화면이다.

#### 5.1.1 구성 요소

**① 상태별 파이 차트**
- 표시: 제정완료(61.4%), 개발중(29.7%), 계획중(4.6%), 제안중(3.7%), 개발중단(0.6%)
- 클릭: 해당 상태의 표준 목록 패널 오픈

**② 12대 전략기술별 수평 막대 차트**
- 표시: 각 기술 분야의 상태별 누적 막대 (제정완료/개발중/제안중/계획중 색상 구분)
- 정렬: 총 건수 내림차순 기본, 클릭으로 전환
- 클릭: 해당 기술 분야의 표준 목록 패널 오픈
- 데이터 예시: 차세대통신(209건), ICT융합(149건), 메타버스(81건) 등

**③ 연도별 제정완료 추이 라인 차트**
- 표시: 2002~2029년 연도별 제정완료 건수 (미래연도는 점선 표시)
- 특정 연도 클릭: 해당 연도 완료 표준 목록 패널 오픈
- 과거 실적과 미래 계획을 같은 선에서 구분하여 표시

**④ 표준기구 Top 5 도넛 차트**
- 표시: ITU-T(341건), 3GPP(74건), ISO/IEC JTC1(66건), IETF(35건), IEC(31건), 기타
- 클릭: 해당 기구의 표준 목록 패널 오픈

**⑤ 직할부서별 현황 히트맵 또는 트리맵**
- 표시: 7개 직할부서 × 표준화 상태의 교차 현황
- 클릭: 해당 부서·상태 조합의 표준 목록 패널 오픈

**⑥ 올해의 변경 현황 카드**
- 표시: 신규 추가(N) 57건 / 갱신(R) 84건
- 클릭: 해당 변경 유형 목록 패널 오픈

> **FRQ-003**: 홈 대시보드는 6개 이상의 차트/위젯을 그리드 레이아웃으로 배치하며, 각 위젯은 독립적으로 클릭·상호작용 가능해야 한다.

### 5.2 전략기술 뷰

12대 전략기술을 기준으로 표준화 현황을 분석하는 화면이다.

#### 5.2.1 구성 요소

**① 기술 분야 선택 탭/버튼**
- 12대 기술 분야 버튼 (전체 포함)
- 활성 기술 분야 시각적 강조

**② 선택된 기술의 표준화 전략 포지션 표시**
- 총 표준 건수(X축) × ETRI 리더십 지표(Y축)의 4분면 매트릭스
  - X축: 해당 전략기술의 총 표준 건수
  - Y축: ETRI 리더십 지표 = (에디터 수 + 의장단 수) / 총 건수 × 100 (%)
    - 에디터·의장단은 다수 참여 시 쉼표 구분으로 각각 인원 수 합산
  - 버블 크기: 총 건수 비례
- 4분면 레이블: 선도(대량·고리더십) / 집중(소량·고리더십) / 참여(대량·성장형) / 소규모(소량·성장형)
- 분면 기준: 각 축의 중앙값(median)으로 동적 분할
- 12개 전략기술을 버블로 동시 표시 (선택 시 하이라이트)
- **클릭: 해당 전략기술의 세부중점기술 드릴다운 패널 인라인 오픈** *(v2.1 변경)*

> **[v2.1 변경 사유]** 이전 버전의 Y축(제정완료율)은 개발 중인 표준이 많은 기술 분야일수록 불리하게 평가되어, 활발한 개발 활동을 반영하지 못하는 문제가 있었다. ETRI 리더십 지표(에디터+의장단 수 비중)로 대체하여 실질적인 표준화 주도력을 반영한다.

**③ 세부중점기술 드릴다운 패널** *(v2.1 신설)*

매트릭스 버블, 상태 분포 바, 또는 상세 테이블에서 전략기술을 클릭하면 해당 기술의 세부중점기술 현황이 인라인으로 펼쳐진다.

구성 요소:
- **헤더**: 선택된 전략기술명, 총 건수, 세부기술 수, "전체 목록 보기" 링크, 닫기 버튼
- **전략 범례**: 선점(보라) / 선도(파랑) / 추격(주황) / 확산(녹색) / 활동없음(회색) 배지
- **세부기술별 행**: 각 행에 다음 정보 표시
  - 세부중점기술명
  - 표준화 전략 배지 (선점/선도/추격/확산 — 보고서 2.3.2절 기준)
  - 활동 비중 (★ 1~5개, 보고서 표2-1 기준)
  - 상태별 미니 누적 바 차트 (제정완료/개발중/제안중/계획중/개발중단)
  - 건수 숫자
  - 마우스 오버 시 주요 표준화 대상(키워드) 툴팁 표시
  - 클릭: 해당 세부기술의 표준 목록 사이드패널 오픈
- **활동 없는 세부기술**: 정적 데이터에만 존재하고 실제 표준 건이 없는 세부기술도 회색으로 표시
- **활동 비중 범례**: ★ 1~5 스케일 설명

데이터 출처:
- 세부기술별 표준 건수 및 상태: Excel 전수조사 데이터의 `subTechArea` 필드 집계
- 전략 배지 (선점/선도/추격/확산): `src/data/techStrategyData.js` 정적 데이터 (보고서 2.3.2절)
- 활동 비중 (★): `src/data/techStrategyData.js` 정적 데이터 (보고서 표2-1)
- 주요 표준화 대상: `src/data/techStrategyData.js` 정적 데이터 (보고서 표2-2~2-13)

**⑤ 참여 표준기구 분포 차트**
- 선택된 기술 분야에서 활용 중인 표준기구 파이/막대 차트
- 클릭: 해당 기구 × 기술 조합 목록 오픈

> **FRQ-004** *(v2.1 수정)*: 전략기술 뷰의 4분면 매트릭스는 X축에 총 표준 건수, Y축에 ETRI 리더십 지표((에디터 수 + 의장단 수) / 총 건수 × 100)를 사용하여 각 전략기술의 좌표를 계산하고 표시해야 한다. 분면 기준은 각 축의 중앙값(median)으로 동적 계산하며 하드코딩을 금지한다.
> **FRQ-004a** *(v2.1 신설)*: 전략기술 클릭 시 해당 기술의 세부중점기술 드릴다운 패널을 인라인 표시한다. 각 세부기술에는 보고서 2.3.2절 기준의 표준화 전략(선점/선도/추격/확산), 표2-1 기준의 활동 비중(★), 상태별 건수 바, 주요 표준화 대상 툴팁을 포함한다. 세부기술 행 클릭 시 해당 표준 목록 사이드패널을 오픈한다.
> **FRQ-005**: 세부기술 집계 테이블에서 빈 셀(0건)은 클릭 불가 상태로 표시한다.

### 6.1 전역 필터 바

모든 뷰에서 상단에 고정 표시되며, 필터 적용 즉시 현재 뷰의 모든 차트와 테이블이 갱신된다.

| 필터 항목 | 유형 | 선택지 |
|----------|------|-------|
| 전략기술 분야 | 다중선택 드롭다운 | 12대 전략기술 + 전체 |
| 표준기구 | 다중선택 드롭다운 | ITU-T, 3GPP, ISO/IEC JTC1, IETF, IEC, ITU-R, ISO, IEEE, 기타 등 |
| 직할부서 | 단일선택 드롭다운 | 7개 부서 + 전체 |
| 표준화 상태 | 다중선택 체크박스 | 제정완료, 개발중, 제안중, 계획중, 개발중단 |
| 완료연도 범위 | 연도 범위 슬라이더 | 2002~2029 |
| 출연처 | 다중선택 드롭다운 | 과기부, 산업부, 보건복지부, 국방부 등 |

> **FRQ-010**: 필터 활성 상태는 각 필터 항목 옆에 배지(Badge)로 표시하고, "전체 필터 초기화" 버튼을 제공해야 한다.  
> **FRQ-011**: 필터 상태는 URL 파라미터에 반영되어 특정 필터 조합을 링크로 공유할 수 있어야 한다.

```

#### 6.2.3 목록 패널 기능

- **엑셀 내보내기**: 현재 목록을 XLSX로 다운로드
- **페이지네이션**: 20건씩 표시, 페이지 이동 또는 무한 스크롤

> **FRQ-012**: 목록 패널의 "컨텍스트 표시" 헤더는 어떤 조건으로 이 목록이 생성되었는지 명확하게 표시해야 한다 (예: "전략기술: 차세대통신 / 상태: 제정완료 / 기구: 3GPP").  
> **FRQ-013**: 목록 패널에서 엑셀 내보내기 시, 현재 표시 중인 필터 조건과 컬럼 구성 그대로 파일이 생성되어야 한다.

### 6.4 검색 기능

#### 6.4.1 글로벌 검색

**검색 대상 필드**: 표준 제목, 표준번호, ETRI 기고자, 주요 내용 (키워드)  
**자동완성**: 입력 2자 이상 시 최대 10건의 표준명 후보 표시

- 매칭 키워드 하이라이트
- "총 N건의 검색 결과" 표시
- 필터와 검색의 AND 조건 적용

> **FRQ-015**: 글로벌 검색은 한글 자모 분리 검색을 지원하지 않아도 되나, 부분 문자열 일치(contains)는 지원해야 한다.

│                                                         │
│                           [ 취소 ]  [ 파일 업로드 → ]    │
└─────────────────────────────────────────────────────────┘
```

> **FRQ-019**: 업로드는 브라우저 파일 선택 다이얼로그와 드래그 앤 드롭 두 가지 방식을 모두 지원해야 하며, .xlsx 파일만 허용한다.  
> **FRQ-020**: 업로드 시 담당자가 갱신 메모를 입력할 수 있어야 한다 (선택). 입력된 메모는 버전 이력에 함께 저장된다.

#### 6.5.2 유효성 검사 (업로드 직후 자동 실행)

파일 파싱 직후, 대시보드가 요구하는 컬럼 구조와 일치하는지 자동 검사한다. 검사는 수초 이내에 완료되며 결과를 화면에 표시한다.

**검사 항목**

| 검사 항목 | 통과 조건 | 실패 시 동작 |
|----------|---------|------------|
| 필수 컬럼 존재 여부 | 27개 컬럼명이 모두 존재 | 오류: 누락 컬럼명 목록 표시 |
| '표준화 상태' 값 범위 | 제정완료·개발중·제안중·계획중·개발중단 外 없음 | 경고: 비정상 값 목록 표시 |
| '(예정) 완료연도' 숫자 형식 | 숫자 또는 공백만 허용 | 경고: 해당 행 목록 표시 |
| '전략기술 분야' 값 범위 | 12대 전략기술 분류 內 | 경고: 신규 분류명 목록 표시 |
| 최소 행 수 | 1건 이상 | 오류: 빈 파일로 간주 |
| 직할부서 컬럼 | 인식 가능한 부서명 포함 | 경고: 신규 부서명 목록 표시 |

**검사 결과 표시 규칙**
- **오류(Error)**: 데이터 적용 불가. 문제를 수정한 뒤 재업로드해야 한다.
- **경고(Warning)**: 데이터 적용은 가능하나, 담당자가 내용을 확인하고 계속 진행할지 선택해야 한다.
- **통과(Pass)**: 다음 단계(변경 미리보기)로 자동 이동한다.

> **FRQ-021**: 유효성 검사에서 오류가 감지된 경우 변경 미리보기로 진행할 수 없으며, 오류 상세 내용(컬럼명, 행 번호, 현재 값)을 제공해야 한다.  
> **FRQ-022**: 경고만 존재하는 경우 "경고를 인지하고 계속 진행" 버튼을 제공하며, 해당 경고 내용은 버전 이력에 기록되어야 한다.

#### 6.5.3 변경사항 미리보기 (Diff 화면)

유효성 검사를 통과하면, 현재 데이터와 업로드된 파일을 자동으로 비교하여 변경사항을 시각화한다.

**레코드 식별 기준**

시스템은 아래 두 필드를 복합키로 사용하여 "같은 표준"인지 판별한다.

```
식별 키 = (표준기구) + (표준번호)
예: "ITU-T" + "Y.1902" → 동일 표준으로 인식
```

식별 키가 없거나 공백인 레코드는 표준 제목을 보조 키로 활용한다.

**변경 유형 분류**

| 변경 유형 | 정의 | 표시 색상 |
|---------|------|---------|
| 신규 추가 (New) | 이전 버전에 없던 식별 키 | 녹색 배경 |
| 내용 수정 (Modified) | 식별 키는 같으나 1개 이상의 필드 값이 변경 | 노란색 배경 |
| 삭제 (Removed) | 이전 버전에 있었으나 새 파일에 없는 식별 키 | 붉은색 배경 |
| 변경 없음 (Unchanged) | 모든 필드가 동일 | 기본 배경 |

**미리보기 화면 구성**

```
┌─────────────────────────────────────────────────────────────┐
│  변경사항 미리보기         [변경된 항목만 보기 ○  전체 보기 ○]   │
├──────────┬──────────┬──────────┬──────────┬─────────────────┤
│ 신규 추가  │  내용 수정  │    삭제    │ 변경 없음  │      전체       │
│  +57건   │   84건   │   3건    │  506건   │    650건        │
├──────────┴──────────┴──────────┴──────────┴─────────────────┤
│ 변경 유형 │ 표준번호 │ 표준 제목        │ 변경 필드         │
├──────────┼──────────┼──────────────────┼────────────────────┤
│ ▲ 신규   │ Y.4900   │ Digital Twin...  │ (신규)            │
│ ✎ 수정   │ Y.1902   │ Framework for... │ 상태: 개발중→제정완료│
│ ✎ 수정   │ TR-0001  │ 6G Framework...  │ 완료연도: 2026→2025│
│ ✕ 삭제   │ H.702    │ Gesture UI...    │ (삭제됨)          │
└──────────┴──────────┴──────────┴────────────────────────────┘
│  [이전으로]              [취소]      [이 내용으로 적용하기 →]  │
└─────────────────────────────────────────────────────────────┘
```

**수정 항목 상세 보기**

목록에서 "내용 수정" 행을 클릭하면 필드별 변경 내용을 인라인 확장하여 표시한다.

```
✎ 수정   Y.1902   Framework for multicast-based IPTV...
  └── 표준화 상태:  [개발중]  →  [제정완료]
  └── 완료연도:     [2024]    →  [2025]
  └── ETRI 기고자:  [강신각]  →  [강신각, 이승익]
```

> **FRQ-023**: 변경 미리보기 화면에서 신규·수정·삭제·변경 없음 탭 필터를 제공하고, 각 탭의 건수를 배지로 표시해야 한다.  
> **FRQ-024**: "내용 수정" 행을 클릭하면 변경된 필드만 이전값(붉은색 취소선) → 신규값(녹색) 형태로 인라인 표시해야 한다.  
> **FRQ-025**: 삭제 항목이 1건 이상인 경우, 적용 확정 버튼에 "삭제 N건 포함" 경고 문구와 추가 확인 모달을 제공해야 한다.

#### 6.5.4 적용 확정 및 즉시 반영

담당자가 미리보기를 확인하고 "적용하기"를 클릭하면 다음이 동시에 실행된다.

1. **스냅샷 저장**: 적용 직전의 현재 데이터를 버전으로 저장
2. **데이터 교체**: 새 파일의 데이터로 대시보드 전체 갱신
3. **기준일 업데이트**: 헤더의 "데이터 기준" 날짜를 오늘 날짜로 변경
4. **갱신 완료 알림**: 화면 상단에 "데이터가 업데이트되었습니다 (신규 +N건, 수정 N건, 삭제 N건)" 토스트 메시지 표시

> **FRQ-026**: 적용이 완료된 직후 대시보드의 모든 KPI 카드, 차트, 테이블은 새 데이터를 반영하여 페이지 새로 고침 없이 즉시 갱신되어야 한다.

#### 6.5.5 버전 이력 관리

과거 업로드 이력과 각 시점의 데이터를 관리하는 기능이다.

**버전 이력 목록**

```
┌─────────────────────────────────────────────────────────────┐
│  버전 이력                                                   │
├──────────┬────────────┬────────────────────────┬────────────┤
│ 버전     │ 적용 일시   │ 갱신 메모              │ 작업       │
├──────────┼────────────┼────────────────────────┼────────────┤
│ v3 현재  │ 2026-07-15 │ 2026년 전수조사 결과   │            │
│ v2       │ 2025-07-22 │ 2025년 전수조사 결과   │ [복원] [비교]│
│ v1       │ 2024-08-10 │ (메모 없음)            │ [복원] [비교]│
└──────────┴────────────┴────────────────────────┴────────────┘
```

**버전별 제공 기능**

- **[복원]**: 해당 버전의 데이터를 현재 데이터로 되돌린다. 복원 전 현재 버전이 자동으로 스냅샷에 저장된다.
- **[비교]**: 해당 버전과 현재 버전의 변경사항을 미리보기 형식으로 표시한다 (읽기 전용, 적용 없음).

**대시보드 내 "이번 갱신 신규·변경 항목" 필터**

전역 필터 바에 "이번 갱신" 토글을 추가한다. 활성화하면 가장 최근 업로드에서 신규 추가되거나 수정된 항목만 표시된다.

> **FRQ-027**: 버전은 최소 5개까지 보관하며, 각 버전은 업로드 파일 전체 데이터를 포함하여 독립적으로 복원 가능해야 한다.  
> **FRQ-028**: "복원" 실행 시 현재 버전이 자동으로 새 스냅샷으로 저장되어 복원 후에도 롤백이 가능해야 한다.  
> **FRQ-029**: 전역 필터의 "이번 갱신" 토글 활성화 시, 목록의 각 행에 "신규" 또는 "수정" 배지를 표시하고, 수정 항목은 상세 패널에서 변경된 필드를 강조 표시해야 한다.

#### 6.5.6 표준 상세 패널 내 변경 이력 탭

개별 표준의 상세 패널에 "변경 이력" 탭을 추가하여, 해당 표준이 과거 버전에서 어떻게 변경되어 왔는지 타임라인으로 표시한다.

```
[기본 정보]  [참여자]  [특허]  [변경 이력 ▶]

  2026-07-15  v3    표준화 상태: 개발중 → 제정완료
                    완료연도: 2026 → 2025

  2025-07-22  v2    ETRI 기고자: 강신각 → 강신각, 이승익
                    세부중점기술: 기타 → 5G 고도화

  2024-08-10  v1    (최초 등록)
```

> **FRQ-030**: 변경 이력 탭은 해당 표준이 1개 이상의 버전에서 수정된 이력이 있을 때만 표시되며, 변경 없는 표준에는 탭 자체를 숨긴다.

#### 6.5.7 갱신 이력 요약 뷰 (홈 대시보드 위젯)

홈 대시보드에 "최근 갱신 현황" 위젯을 추가한다.

```
┌─────────────────────────────────────────┐
│  최근 갱신 현황             2026-07-15  │
├─────────────────────────────────────────┤
│  신규 추가     +57건    [목록 보기 →]   │
│  내용 수정      84건    [목록 보기 →]   │
│  삭제            3건    [목록 보기 →]   │
│  전년 대비      +54건 증가              │
└─────────────────────────────────────────┘
```

> **FRQ-031**: 홈 대시보드의 갱신 현황 위젯에서 "목록 보기" 클릭 시, 해당 변경 유형으로 필터링된 표준 목록 패널을 즉시 오픈해야 한다.

## 7. 데이터 요구사항

### 7.1 데이터 소스 및 처리

| 항목 | 내용 |
|-----|------|
| **원본 데이터** | ETRI 표준화 현황 전수조사 Excel (650건, 27컬럼) |

| **갱신 주기** | 연 1회 전수조사 기준 (담당자 수동 업로드) |
| **데이터 식별 키** | 표준기구 + 표준번호 복합키 (보조: 표준 제목) |
| **버전 보관 수** | 최근 5개 버전 (각 버전 = 전체 데이터 스냅샷) |

| **저장 방식** | 브라우저 IndexedDB (소규모) 또는 서버 파일시스템 |

### 7.2 데이터 상태 모델

시스템은 다음의 4가지 데이터 상태를 관리한다.

```
[활성 데이터 (Active)]
    현재 대시보드에 표시되는 최신 버전 데이터.
    필터·검색·집계의 기준이 된다.

[스냅샷 (Snapshot)]
    각 업로드 적용 시점의 전체 데이터 사본.
    버전 복원 및 비교에 사용.
    버전 이력: v1, v2, v3 … (최대 5개)

[변경 집합 (Changeset)]
    두 버전 간 diff 결과 (신규/수정/삭제/변경없음).
    미리보기 화면과 변경 이력 탭에 사용.
    각 스냅샷 전환 시 자동 계산하여 함께 저장.

[갱신 메타 (Update Meta)]
    업로드 일시, 담당자 메모, 유효성 경고 내역.
    버전 이력 목록에 표시.
```

### 7.3 데이터 컬럼 매핑

| 원본 컬럼명 | 시스템 내 필드명 | 용도 |
|-----------|----------------|------|
| 소 | institute | 직할부서 필터 |
| 본부(단) | department | 부서 뷰 |
| 표준 제목 (예정) | title | 목록·검색·상세·diff 비교 |
| 표준번호 | stdNo | **식별 키 (1)**, 목록·상세 |
| 표준기구 | stdBody | **식별 키 (2)**, 기구 뷰·필터 |
| 위원회, 작업반 등 | workingGroup | 기구 상세 드릴다운 |
| 표준화 상태 | status | KPI·필터·목록·diff |
| 표준 주요 내용 | abstract | 상세 패널 |
| (예정) 시작년도 | startYear | 연도 뷰 |
| (예정) 완료연도 | endYear | 연도 뷰·타임라인·diff |
| ETRI 기고자 | contributor | 목록·상세·연관탐색·diff |
| ETRI 에디터(예정포함) | editor | 목록·상세 |
| ETRI 의장단 | chairPosition | 집계 |
| 사업분류 | projectType | 상세 |
| 출연처 | fundingAgency | 필터 |
| 전략기술 분야 | techArea | 전략기술 뷰·필터·diff |
| 세부중점기술 분야 | subTechArea | 세부기술 집계 |
| 표준특허 유무 | hasPatent | 특허 뷰·필터 |
| 표준특허 개수 | patentCount | 특허 집계 |
| 특허상태(출원/등록) | patentStatus | 특허 상세 |
| 변경여부 | changeTypeRaw | 신규(N)/갱신(R) — 원본 엑셀 표기 |

**시스템 생성 필드 (원본 엑셀에 없음)**

| 필드명 | 생성 시점 | 내용 |
|-------|---------|------|
| `_recordKey` | 데이터 로딩 시 | `stdBody + ":" + stdNo` (식별 키) |
| `_changeType` | diff 계산 시 | `new` / `modified` / `removed` / `unchanged` |
| `_changedFields` | diff 계산 시 | 수정된 필드명 목록 및 이전값·신규값 |
| `_versionAdded` | 스냅샷 저장 시 | 이 레코드가 처음 등장한 버전 번호 |
| `_lastModified` | 스냅샷 저장 시 | 이 레코드가 마지막으로 수정된 버전 번호 |

### 7.4 파생 데이터 (계산 필드)

| 파생 필드 | 계산 방법 | 활용 |
|----------|----------|------|
| ETRI 리더십 지표 | (에디터 수 + 의장단 수) / 총 건수 × 100 | 전략기술 뷰 Y축 *(v2.1 신설)* |
| 전략 포지션 | 총건수(X)·리더십지표(Y) 사분면 | 4분면 레이블 *(v2.1 수정)* |
| 연도 구분 | 완료연도 ≤ 현재연도: 실적 / > 현재연도: 계획 | 연도 뷰 시각화 |

> **주의**: 연도 구분의 기준 연도는 하드코딩하지 않고 시스템 현재 연도를 동적으로 참조한다.

### 7.6 정적 참조 데이터 *(v2.1 신설)*

보고서에서 추출한 정적 데이터를 `src/data/techStrategyData.js`에 관리한다.

| 데이터 | 출처 | 내용 |
|--------|------|------|
| 12대→42대 매핑 | 보고서 1.3절 | 전략기술별 세부중점기술 분류 |
| 표준화 전략 | 보고서 2.3.2절 | 세부기술별 선점/선도/추격/확산 배정 |
| 활동 비중 | 보고서 표2-1 | 세부기술별 ★ 1~5 등급 |
| 주요 표준화 대상 | 보고서 표2-2~2-13 | 세부기술별 표준화 키워드 목록 |

> **주의**: 보고서 개정 시 정적 데이터도 함께 갱신해야 한다. 전략 배정 및 활동 비중은 보고서 버전에 종속된다.

### 7.5 데이터 저장 구조 (개념)

```
로컬 저장소 (IndexedDB 또는 서버)
├── versions/
│   ├── v1/
│   │   ├── data.json       ← 전체 레코드 배열
│   │   ├── meta.json       ← { date, memo, warnings, totalCount }
│   │   └── changeset.json  ← diff vs 이전 버전
│   ├── v2/ ...
│   └── v3/ (active)
└── config.json             ← { activeVersion: "v3", maxVersions: 5 }
```

## 8. 인터랙션 설계 원칙

### 8.1 드릴다운 네비게이션 흐름

모든 사용자 인터랙션은 다음의 일관된 패턴을 따른다.

```
[차트/테이블 내 수치 클릭]
         ↓
[사이드 패널: 표준 목록 슬라이드인]
  - 컨텍스트 헤더 표시
  - 필터된 표준 목록 테이블
         ↓
[목록에서 개별 표준 클릭]
         ↓
[패널 내용 교체: 표준 상세 정보]
  - 뒤로 가기 버튼으로 목록 복귀
         ↓
[연관 탐색 버튼 클릭] (선택)
         ↓
[패널 내용 교체: 다른 조건의 목록]
  - 뒤로 가기 스택 유지
```

### 8.2 상태 관리 원칙

> **FRQ-016**: 브라우저 뒤로 가기 버튼은 패널 내 탐색 히스토리를 단계별로 복원해야 한다.  
> **FRQ-017**: 전역 필터 변경 시, 현재 열려 있는 목록/상세 패널도 해당 필터를 반영하여 즉시 갱신되어야 한다.  
> **FRQ-018**: 차트 클릭으로 오픈된 목록 패널에서 적용된 컨텍스트 조건(예: "차세대통신 > 제정완료")은 전역 필터 바에도 시각적으로 반영되어야 한다.

## 9. 비기능 요구사항

### 9.1 성능

| 항목 | 요구사항 |
|-----|---------|
| 초기 페이지 로드 | 3초 이내 (650건 전체 데이터 포함) |
| 차트 렌더링 | 필터 변경 후 1초 이내 갱신 |
| 패널 슬라이드 애니메이션 | 300ms 이내 완료 |
| 검색 자동완성 | 입력 후 200ms 이내 후보 표시 |

### 9.2 반응형 지원

| 디바이스 | 레이아웃 변경 |
|---------|-------------|
| 데스크톱 (1280px 이상) | 메인 + 사이드 패널 2컬럼 레이아웃 |
| 태블릿 (768px~1279px) | 메인 단일 컬럼, 패널 오버레이 방식 |
| 모바일 (767px 이하) | 최소 지원 (주요 KPI 카드 및 목록만 표시) |

### 9.3 접근성

- 모든 차트는 데이터 테이블로 대체 접근 가능 (시각 장애 대응)
- 색상만으로 상태를 구분하지 않고 아이콘 또는 텍스트 레이블 병기
- 키보드만으로 전체 탐색 가능 (Tab, Enter, ESC)

### 9.4 브라우저 호환성

- Chrome 최신 2개 버전
- Edge 최신 2개 버전
- (모바일) iOS Safari, Android Chrome
